## 1. CONVENTIONS <hr>
En Java, 
- 1 classe = 1 fichier 
- 1 dossier = 1 package

Package = un espace de nom (namespace)
ex. namespace: `com.societe.projet`

namespace toujours en minuscule
nom de class tjs en Camel Case (MajusculePourChaqueNouveauMot)

Camel Case : pour le nom des classes  (LeJoliPoney)
Lower Camel Case: seulement la premiere lettre (leJoliPoney)

Pour les fonctions et les variable :
- On met pas de _ ou de -, pas de char speciaux, pas de point, pas d'espace.
- Les nombres sont fortement deconseilles.

## 3. MODELE MVC
Modele MVC : Model View Controller

Model :
- c'est la representation des donnees dans le logiciel pour cette appli
- representation d'un objet du reel pour notre appli (ex: Arbre, User)
- C'est la representation des grandds objets

View : 
- ce que user voit (GUI: graphic user interface). 
- moteurs graphiques: Swing (Java), FX (Java), Xaml (C#), etc.
- HTML, CSS, JS
- JS Angular pour vue
- on va utiliser Freemarker qui va generer du html/css

Controller:
- c'est un element du code
- c'est le code metier de mon app
- permet de gerer une interaction specifique du code metier
- ex: les forestiers de bretagne veulent elaguer toutes les routes qui ont des branches d'arbres qui depassent de plus d'1 metre
    - c quoi un arbre, c quoi une route, c quoi depasser d'1m ...
- GUI: route a selectionner + arbre sur la route a selectionner + une option de taille + bouton valider
- controller gere une vue ou un groupe d'interaction

un element du model est incapable d'interagir avec l'exterieur

c'est le controlleur qui va interagir pour lui

**Notes**:
- Django: web framework, model-view-template
- Possible de se passer du HTML, CSS etc avec des outils qui convertissent directement en Java
- FreeMarker pour generer du html/css
